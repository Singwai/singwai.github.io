The article is intented to demonstrate how is this website boostraped using Hugo. This blog is powered by a open source project named Hugo. Hugo is a static site generator written by Go. The project is well supported with 40.6k star and 4.6k fork as of early 2020. The article will provide insturction all the way from installation, pick a theme to deployment to AWS S3 manually.

Install Hugo

If you are running macOS, you can install Hugo simplily using brew.

brew install hugo

If you are running other operating system, you can use this see the install instruction here. https://gohugo.io/getting-started/installing/

Initialize Project

hugo new site {{site_name}}
cd {{site_name}}
git init

We also want to initialize the project with git because we install a theme using git submodule. git submodule is a way to mange a git repo inside a git repo. See the link for more information. https://git-scm.com/book/en/v2/Git-Tools-Submodules

Pick a Theme

After the project is set up, you can pick a theme here and I picked github theme. https://themes.gohugo.io/

Install Theme

git submodule add git@github.com:MeiK2333/github-style.git themes/github-style

In the config.toml file, you need to pick the theme. I also supplied some parameters that the theme is needed

languageCode = "en-us"
title = "My New Hugo Site"
theme = "github-style"

[params]
  author = "Singwai Chan"
  description = "A software engineer"
  github = "singwai"
  email = "c.singwai@gmail.com"
  utterances = "example/example.github.io"
  url = "https://singwaichan.com"

Build and develop

You can generate a page using the following command.

hugo new posts/hello-world-hello-hugo

This will create and markdown file with some default configuration under content/posts. You can compose the article in the markdown file.

# Build and Develop 
# D will print out debug message when change is detected and/or error.
hugo server -D

This will start the server and start listening at port 1313. You should able to see your page at localhost:1313
